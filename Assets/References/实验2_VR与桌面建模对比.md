# 实验2：VR建模 vs 传统桌面建模对比

## 一、研究目标

对比VR笔刷建模与Blender曲线拉伸在效率、精度和用户偏好上的差异。

## 二、实验设计

### 2.1 对比工具
**工具A：VRBrush（VR环境）**
- Quest 2头显 + 手柄交互
- 沉浸式3D空间绘制

**工具B：Blender（桌面环境）**
- PC + 鼠标键盘
- 曲线拉伸工具

### 2.2 测试任务（3项）

首先他们需要掌握Blender和ADBrush系统中的绘制方法。这个过程不限时也不计时，但是在任务完成后会有一个主观评价。我们可以使用实验1中所涉及到的人员。

**任务1：简单结构**
- 要求：创建圆柱体侧面。

**任务2：螺旋结构**
- 要求：创建螺旋弹簧形状

**任务3：形变管道**
- 要求：创建由圆形转为方形的管道。具体来讲，该管道有一个九十度的弧形过渡的夹角，夹角的一段为圆形管道，另一端靠近夹角的位置为圆形，远离夹角的位置为方形管道

### 2.3 参与者
- 人数：20名
- 要求：同实验1，注意我们会询问用户是否使用过Blender。
- 顺序：随机分配先用VR或Blender
- 时间：该实验可能会消耗巨量的时间。对于Blender，用户可能无法完成学习任务.此时考虑替代方案。

## 三、数据收集

### 3.1 定量指标

**客观测量**
- 任务完成时间（秒）

**用户评分（10分制）**
1. **操作效率**：完成任务的速度感受
2. **控制精度**：精确表达设计意图的能力
3. **学习难度**：掌握工具所需时间和精力
4. **整体偏好**：更愿意使用哪个工具

### 3.2 开放式问题

**开放式问题**
- 两种工具各自的优势和局限是什么？
- 在什么场景下您会选择VR或桌面软件？

## 四、评估方法

### 4.1 定量分析
**统计检验**
- 配对t检验：比较两工具在完成时间、评分上的差异
- 描述统计：各指标的均值、标准差

### 4.2 定性分析
**主题分析**
- 对开放式回答进行编码
- 总结两种工具的适用场景

## 五、实验流程

**Session A（VR先行组，约90分钟）**
1. VR系统培训（15分钟）
2. VR任务执行（30分钟）
3. 休息（10分钟）

4. Blender培训（90分钟）

5. Blender任务执行（30分钟）
6. 问卷和访谈（15分钟）

**Session B（Blender先行组）**
- 流程相同，顺序相反

## 六、预期成果和缺陷

- 定量数据：两工具在效率、精度、偏好上的差异
- 缺陷：我们很难招募到专业的Blender用户。因此我们面向的受试者大多是不熟悉Blender的用户。如果使我们可能会招募更多的用户来单独进行实验2.
